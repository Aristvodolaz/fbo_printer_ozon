# ⚠️ Внимание: Порт 22

## Важная информация

Порт **22** обычно используется для **SSH** (Secure Shell) - это стандартный порт для удаленного доступа к серверу.

## Потенциальные проблемы

1. **Конфликт с SSH:** Если на сервере запущен SSH сервер, порт 22 будет занят
2. **Требуются права root:** Для использования портов ниже 1024 обычно нужны права администратора
3. **Безопасность:** Использование стандартных портов может быть небезопасно

## Решения

### Вариант 1: Использовать другой порт (рекомендуется)

Запустите с другим портом через переменную окружения:

```bash
PORT=3022 pm2 start server.js --name fbo-printer-ozon
```

Или измените в `server.js`:
```javascript
const PORT = process.env.PORT || 3022;
```

### Вариант 2: Остановить SSH (не рекомендуется)

Если вы уверены, что SSH не нужен:
```bash
sudo systemctl stop sshd
```

**⚠️ ВНИМАНИЕ:** Это отключит удаленный доступ к серверу!

### Вариант 3: Использовать порт выше 1024

Рекомендуемые порты:
- `3022` - похож на 22, но свободен
- `8080` - стандартный для веб-приложений
- `3000` - популярный для Node.js
- `3019` - был использован ранее

## Текущая конфигурация

Сейчас сервер настроен на порт **22**.

Если возникнут проблемы при запуске, используйте:
```bash
PORT=3022 npm start
```

Или для PM2:
```bash
PORT=3022 pm2 start server.js --name fbo-printer-ozon
```
